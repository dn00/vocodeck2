## Envelope

Every event is one JSON object:

```json
{"v": 1, "seq": 42, "ts": 1751500000.0,
 "type": "...", "payload": {}}
```

- `seq` is global and monotonic per daemon run; there is NO replay —
  reconnect and read the fresh `snapshot`.
- Consumers MUST ignore unknown types and unknown payload fields
  (additive evolution).
- Wire values use role names, never model names (`first_mate`, not a
  model name).

## Surfaces (localhost only; remote via SSH tunnel)

- `GET  /v1/events` — WebSocket. First message is a `snapshot` event;
  live events follow. Commands may be sent as
  `{"id": "...", "cmd": "<command>", "payload": {}}`; replies are
  `{"id": "...", "ok": true, "payload": {}}` or
  `{"id": "...", "ok": false, "error": "..."}`.
- `POST /v1/control/<command>` — same commands over HTTP.
- `POST /v1/bridge/register|say|screen`, `GET /v1/bridge/listen` — the
  agent bridge (see SPEC §8.1 for exact payloads and fail-soft rules).
