Policies (written down, so they survive)

  • Destructive actions: undo over confirm where reversible — withdraw annotation and detach agent get an "— undo" toast (detach never touches the process). Confirm is reserved for kill (session or terminal): it is irreversible and says what dies.
  • Toasts: errors persist until dismissed and carry the server's contextual message (the protocol fix guarantees one exists); successes auto-fade in 4s. Never more than one toast per event.
  • Accessibility floor: captions and transcript are aria-live="polite" regions; state is never color-only (every dot has an adjacent word); micro text is ink2 (≥4.5:1 contrast), ink3 never sets words; every focusable shows a visible ring.
  • Keyboard floor (U1 tasks): Esc closes modals · Enter submits · focus ring everywhere · PTT hold key while the browser has focus. Ctrl+P quick-switcher stays parked.
  • Motion: orb pulse, listening bars, speaking eq — all voice, nothing else animates. prefers-reduced-motion stills all three.

Decisions already made (unchanged)

  • Scoping: one selection (the agent) drives pages, annotations, transcript, routing. Voice moments are the single global exception. The dock header names its scope.
  • Forks (user, 2026-07-07): orb click-cycles + hold-PTT · input in the top strip · transcript bounded 50/side · diff smart auto-expand + rail file sub-tree.
  • Naming: dock tab annotations (lists things); repo button review (starts an activity).
  • Workspace is data-only; the rail groups agents by repo (common_dir); agentless groups are review-only.
  • Daemon budget (all of it): page.publish · workspace.open · speech.started/finished gain who+text · speech.sentence per-sentence progress (emitted from _sentence_synth's generator; degrade = whole-message highlight) · per-session user-input log symmetric to say_log. Spawn/connect need nothing new.
