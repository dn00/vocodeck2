Naming: "annotations", with one exception. The dock tab is now ANNOTATIONS — it lists things (concerns, questions, replies), and that's the diff-annotate lineage. Agent status tracking (herdr comparison, verified)

  • herdr: process-name matching + terminal-output heuristics, no hooks; four aggregate states (blocked/working/done/idle) with sidebar color icons; notifications configurable.
  • deck: strictly stronger ground truth — the bridge is a fact source heuristics can't beat: an agent parked on listen is provably listening, a dispatch is provably working, a pending ask is provably blocked-on-you. §6's display_state() derives from bridge facts first and falls back to the same pane-watcher heuristics herdr uses (ANSI-stripped classify) only for unmanaged terminals. What we adopt from herdr is the presentation: aggregate counts in the status line, color-block state dots, blocked-first rail ordering. What we lack (parked): state-change notifications.

Why these calls (the two you asked about)

  • Tabs → rail tree. Horizontal tabs were a second navigation axis duplicating the rail's selection, and they needed the agent's name in every label to stay unambiguous. Nesting pages under the agent makes "one agent → one surface" structural: selecting Freya is opening her surfaces; page labels drop the name because the tree already says whose they are. The center header carries context (FREYA / diff · vs origin/main · r2) plus the page-level actions that used to have no home (expand all, ⊕ review). If page counts grow, a Ctrl+P quick-switcher slots in without changing the IA.
  • Transcript, not chat. The messenger feel came from bubbles + alternation + a dedicated input. All three are gone: flat log entries, one shared column, and the ONLY input stays in the presence strip. What remains is what a voice-first app genuinely needs — a captions history you can scroll and quote. If even the tab feels too chatty later, the same log can render as an overlay on ` (backtick), but the dock tab is the better default: it sits beside the review ledger it gives context to.

Unchanged from rev 2 (already agreed)

  • Scoping model: one selection (agent) drives pages, ledger, transcript, routing; voice moments are the single global exception; dock header names the scope.
  • Daemon additions (the whole budget): page.publish, workspace.open, speech payload enrichment (who + text) + the per-sentence speech.sentence progress event (rev 3.2 — see the karaoke supportability row), per-session user-input log symmetric to say_log (deque(50), persisted, recorded at dispatch; route/moment data never stored). The spawn and connect modals need nothing new — session.spawn and the adapters already exist.
  • Annotation editor: diff-annotate reference structure verbatim (target label, textarea, Concern|Question|Nit + blocking, Add annotation/Cancel, tip line) — skin is now brutalist.
  • Build order U0–U3, each slice ending at your click-through.

Open forks — your call before U1

  • Orb interaction: click cycles attention + hold = PTT (proposed), vs click = PTT toggle + menu.
  • Input placement: top strip (shown) vs single bottom bar — one exists either way.
  • Transcript depth: bounded per-agent log (50/side, restart-surviving — proposed) vs fuller durable transcript. Lean: bounded.
  • New — collapsed-diff default scope: collapse everything except files with open findings / changed-since-your-rev (shown), or collapse strictly everything. Lean: the former — the reason you opened the diff should already be visible.
